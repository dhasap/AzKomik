# .gitignore Documentation

File ini menjelaskan mengapa beberapa file/folder di-ignore dalam repository.

### 1. `gambar-referensi/`
**Alasan**: Berisi gambar desain referensi UI/UX yang bersifat pribadi/konfidensial.
- File desain dari Figma, Sketch, Adobe XD
- Screenshot referensi dari aplikasi lain
- Asset gambar yang tidak untuk distribusi publik

### 2. `tachiyomiJ2K/`
**Alasan**: Source code atau file referensi dari project TachiyomiJ2K.
- Mungkin berisi code yang tidak boleh di-redistribusi
- File studi/referensi implementasi
- Dokumentasi internal

### 3. `docs-private/`
**Alasan**: Dokumentasi pribadi yang tidak untuk publik.
- Catatan pengembang
- TODO list pribadi
- Dokumentasi API internal
- Planning dan roadmap yang belum final

### 4. `.env` & `*.properties` (konfigurasi)
**Alasan**: Berisi informasi sensitif.
- API keys
- Database credentials
- Konfigurasi environment lokal
- Secret tokens

### 5. `signing/` & `keystore files`
**Alasan**: Keamanan - JANGAN PERNAH commit keystore!
- File `.jks`, `.keystore`, `.p12`
- Password dan alias
- Certificate files

## ğŸ”’ Keamanan

### Jangan pernah commit:
1. **Keystore files** - Bisa digunakan untuk sign APK berbahaya
2. **API Keys** - Bisa disalahgunakan
3. **Google Services JSON** - Berisi credential Firebase
4. **File konfigurasi produksi** - Bisa expose server details

### Best Practices:
- Gunakan `local.properties` untuk konfigurasi lokal
- Gunakan GitHub Secrets untuk CI/CD
- Gunakan `.env.example` sebagai template (tanpa nilai asli)

## â“ FAQ

**Q: Kenapa `gambar-referensi/` di-ignore?**  
A: Karena berisi desain UI dari aplikasi lain yang mungkin memiliki hak cipta, dan tidak perlu dilihat publik.

**Q: Bagaimana cara menambahkan file ke ignore?**  
A: Edit `.gitignore` dan tambahkan pola file/folder yang ingin di-ignore.

**Q: Apa yang terjadi jika saya sudah commit file yang seharusnya di-ignore?**  
A: Gunakan:
```bash
git rm --cached filename
git commit -m "Remove file that should be ignored"
```
